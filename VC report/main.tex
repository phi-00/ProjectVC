\begin{abstract}
This report presents the first phase of a deep learning project focused on traffic sign recognition using the German Traffic Sign Recognition Benchmark (GTSRB) dataset. The main objective of this phase was to study the impact of different data augmentation techniques and image preprocessing strategies on the performance of convolutional neural networks (CNNs). Several augmentation configurations were evaluated, including geometric transformations, color-based augmentations, combined augmentations, and massive dataset expansion. Experimental results demonstrate that data augmentation significantly improves model generalization and robustness, achieving a maximum average test accuracy of approximately 98.38\%.
\end{abstract}

\begin{IEEEkeywords}
Deep Learning, Computer Vision, Traffic Sign Recognition, GTSRB, Convolutional Neural Networks, Data Augmentation, Image Processing, Ensemble Learning
\end{IEEEkeywords}

\section{Introduction}
Traffic sign recognition is an important task in computer vision and autonomous driving systems. Modern Advanced Driver Assistance Systems (ADAS) rely heavily on robust visual recognition models capable of detecting and classifying traffic signs under different environmental conditions such as illumination changes, rotations, occlusions, and motion blur.

The objective of this project is to explore deep learning techniques applied to the GTSRB dataset in order to achieve the highest possible classification accuracy. According to published benchmarks, the best reported accuracy on the dataset is approximately 99.82\%.

In the first phase, several image preprocessing and data augmentation techniques are evaluated. The goal is to understand how different augmentation strategies influence model generalization and robustness.

In the second phase, ensemble learning techniques are explored. Multiple models trained during Phase 1 are combined using voting strategies in order to reduce prediction variance and improve overall classification performance.

\section{Dataset}

The German Traffic Sign Recognition Benchmark (GTSRB) is a well-known benchmark dataset for traffic sign classification.

The dataset contains:
\begin{itemize}
    \item 43 traffic sign classes;
    \item More than 50,000 images;
    \item Images captured under real-world conditions;
    \item Significant variability in illumination, perspective, scale, and occlusion.
\end{itemize}

The dataset was divided into:
\begin{itemize}
    \item Training set;
    \item Validation set;
    \item Test set.
\end{itemize}

\subsection{Class Distribution}

One important characteristic of the dataset is the imbalance between classes. Some traffic signs contain considerably more samples than others, which can introduce bias during training.

The initial dataset analysis revealed:
\begin{itemize}
    \item Strong variability in the number of samples per class;
    \item Variability in image resolution;
    \item Presence of noisy samples and illumination changes.
\end{itemize}

\section{Model Architecture}

Three different convolutional architectures were evaluated.

\subsection{Custom CNN (Conv\_II)}

A custom convolutional neural network was developed specifically for the GTSRB dataset. The architecture includes:

\begin{itemize}

\item Max-pooling operations;
\item Fully connected classification layers;

This architecture was designed to balance computational efficiency and classification accuracy.

\subsection{VGG16}

The second architecture corresponds to the classical VGG16 model trained from scratch on the GTSRB dataset.

\subsection{VGG16 with Transfer Learning}

A third model was obtained using transfer learning from ImageNet-pretrained VGG16 weights. Feature extraction layers were frozen and only the classifier layers were retrained on the traffic sign dataset.

\subsection{Training Strategy}

All experiments used the same training configuration to ensure a fair comparison between augmentation techniques.

The training pipeline included:

\begin{itemize}

\item Model checkpointing based on validation performance.

To reduce the influence of random initialization, each configuration was trained multiple times and the average test accuracy was recorded.

\section{Phase 1 - Data Augmentation Strategies}
The first phase is focused on evaluating the impact of data augmentation techniques on model performance.

Data augmentation is a key technique to improve the generalization ability of deep learning models, especially in image classification tasks such as traffic sign recognition. In this work, multiple augmentation strategies were systematically designed and evaluated using the GTSRB dataset.

The main objective was to analyze how different transformations affect model robustness and accuracy under real-world variations such as rotation, illumination changes, occlusions, and perspective distortion.

All experiments follow a controlled evaluation protocol using the same CNN architecture, optimizer (Adam), and training configuration, varying only the augmentation pipeline.

\subsection{Experiment 0 - Baseline}

The baseline experiment uses only minimal preprocessing, without any augmentation applied to the training data. This serves as the reference for all subsequent comparisons.

\begin{lstlisting}[language=Python]
    _, mean, std = run_experiment(
        name='baseline',
        transform_train=t_base,
        transform_test=t_base,
        model_name=model_name
    )
\end{lstlisting}

\subsection{Experiment 1 - Geometric Augmentation}

This experiment introduces basic geometric transformations, aiming to improve invariance to small spatial changes such as rotation and translation.

\begin{lstlisting}[language=Python]

transform_geo = v2.Compose([
    v2.RandomRotation(15, interpolation=v2.
    InterpolationMode.BILINEAR),
    v2.Resize((IMG_SIZE_CONV, IMG_SIZE_CONV)),
    v2.RandomAffine(degrees=0, translate=(0.1, 0.1)),
    v2.ToImage(),
    v2.ToDtype(torch.float32, scale=True)
])
\end{lstlisting}

These transformations simulate small camera shifts and orientation changes commonly found in real driving scenarios.

\subsection{Experiment 2 - Advanced Geometric Augmentation}

A more aggressive geometric pipeline was introduced using perspective distortion and random cropping. This aims to simulate more extreme viewpoint variations.

\begin{lstlisting}[language=Python]
transform_geo2 = v2.Compose([
    v2.RandomPerspective(distortion_scale=0.2, p=0.6),
    v2.Resize((IMG_SIZE_CONV +8, IMG_SIZE_CONV +8)),
    v2.RandomCrop((IMG_SIZE_CONV, IMG_SIZE_CONV)),
    v2.ToImage(),
    v2.ToDtype(torch.float32, scale=True)
])
\end{lstlisting}

This setup introduces stronger spatial variability while maintaining the semantic structure of traffic signs.

\subsection{Experiment 3 - Color Augmentation}

This experiment focuses on photometric transformations using ColorJitter operations. Brightness, contrast, saturation and hue were randomly modified to simulate different illumination and weather conditions.

\begin{lstlisting}[language=Python]
transform_color = v2.Compose([
    v2.Resize((IMG_SIZE_CONV, IMG_SIZE_CONV)),
    v2.ColorJitter(brightness=0.5, contrast=0.5,
    saturation=0.3, hue=0.05),
    v2.ToImage(),
    v2.ToDtype(torch.float32, scale=True)
])
\end{lstlisting}

Color jittering is particularly relevant for real-world driving scenarios where illumination is highly variable.

\subsection{Experiment 4 - Combined Augmentation}

This experiment combines geometric and color augmentations, along with random erasing to simulate occlusions such as dirt, shadows, or partial obstruction.

\begin{lstlisting}[language=Python]
transform_combined = v2.Compose([
    v2.RandomRotation(15, interpolation=v2.
    InterpolationMode.BILINEAR),
    v2.Resize((IMG_SIZE_CONV, IMG_SIZE_CONV)),
    v2.RandomAffine(degrees=0, translate=(0.1, 0.1)),
    v2.ColorJitter(brightness=0.5, contrast=0.5,
    saturation=0.3, hue=0.05),
    v2.ToImage(),
    v2.ToDtype(torch.float32, scale=True),
    v2.RandomErasing(p=0.3, scale=(0.02, 0.1),
    value='random')
])
\end{lstlisting}

Random erasing improves robustness by forcing the network to learn partial-feature representations.

\subsection{Experiment 5 - Massive Dataset Expansion}

This experiment significantly increases dataset diversity by duplicating the training dataset with a strongly augmented version. The final dataset is a concatenation of original and augmented samples.

transform_strong = v2.Compose([

\subsection{Experiment 6 - MixUp Augmentation}

This experiment uses MixUp, a data augmentation technique that generates synthetic training samples by interpolating pairs of images and their labels.

\begin{lstlisting}[language=Python]

Unlike standard training, MixUp produces soft labels, improving decision boundaries and reducing overfitting.

During evaluation, standard validation is performed without MixUp to ensure fair performance measurement.

\section{Phase 1 Results and Discussion}

The experimental results demonstrate that data augmentation has a significant influence on classification performance.

Several observations can be highlighted:

\begin{itemize}
\item Baseline models already achieved strong performance due to the relatively large size and quality of the GTSRB dataset.

\item Moderate geometric transformations improved robustness to viewpoint changes, although excessively aggressive transformations occasionally degraded performance by producing unrealistic samples.

\item Photometric augmentation improved robustness to illumination variations while maintaining stable accuracy.

\item Combined augmentation increased training diversity but also introduced additional optimization difficulty.

\item Massive dataset expansion consistently produced some of the strongest results, indicating that increasing sample diversity is highly beneficial for traffic sign recognition.

\item MixUp regularization improved generalization by encouraging smoother class boundaries and reducing overfitting.

\end{itemize}

Overall, the results confirm that augmentation strategies contribute positively to model robustness and generalization.

\section{Phase 2 – Ensemble Learning}

Although individual neural networks achieved strong performance, different models tend to make different classification errors. Ensemble learning exploits this diversity by combining multiple predictors into a single decision system.

The objective of Phase 2 was therefore to evaluate whether combining models trained during Phase 1 could further improve classification accuracy.

\subsection{Ensemble Construction}

For each augmentation strategy, the models obtained from different training runs were combined into an ensemble.

Let

\[
M = \{m_1, m_2, \dots, m_n\}
\]

represent the set of trained models.

Two ensemble aggregation methods were investigated.

\subsection{Soft Voting}

In Soft Voting, the posterior probabilities predicted by all models are averaged.

Given a class $c$, the ensemble probability is computed as

\[
P(c) = \frac{1}{n}\sum_{i=1}^{n} P_i(c),
\]

where $P_i(c)$ is the probability assigned by model $m_i$.

The final prediction corresponds to the class with maximum averaged probability.

Soft Voting exploits confidence information from all classifiers and is generally expected to provide better performance when the individual models are well calibrated.

\subsection{Hard Voting}

In Hard Voting, each model independently predicts a class label.

The final prediction is obtained through majority voting:

\[
\hat{y} =
\operatorname*{arg\,max}_{c}
\sum_{i=1}^{n}
\mathbf{1}(y_i=c),
\]

where $y_i$ is the prediction of model $m_i$.

This strategy ignores confidence scores and only considers final class decisions.

\end{itemize}
